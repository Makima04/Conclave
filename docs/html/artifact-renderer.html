Artifact Renderer 规范

定义 LLM 生成的 UI 内容如何安全、高效地渲染：三层渲染模型、Artifact 生命周期、UI Schema、iframe 沙箱、资源预算和快照策略。

目标

  • LLM 输出不直接注入主聊天 DOM，所有 UI 变化通过受控渲染通道。
  • 支持从简单状态更新到复杂自定义 UI 的渐进式渲染能力。
  • 每个渲染单元有独立的资源预算，长会话不会因 DOM 堆积而卡顿。
  • 离屏渲染单元可安全卸载并恢复，不影响用户体验。
  • 自定义代码在 iframe 沙箱中运行，无法访问主页面 DOM、存储或网络。

三层渲染模型

第一层

数据驱动 UI

LLM 输出结构化 state diff，平台用内置组件渲染。

适用：物品增减、属性变化、任务状态更新、关系变化。

安全性：最高。无代码执行，纯数据更新。

第二层

主题化组件

LLM 声明 renderer + theme + props，平台校验后渲染。

适用：奇幻物品卡片、角色面板、地图标记、好感度组件。

安全性：高。渲染器是平台内置的，LLM 只填数据。

第三层

自定义 Artifact

LLM 提供 HTML/CSS/JS，在 iframe 沙箱中运行。

适用：小游戏、复杂交互面板、独特角色 UI。

安全性：需要权限声明和资源预算控制。

层级选择

场景推荐层LLM 输出是否允许代码
新增普通物品第一层state_diff否
新增带图物品第二层component_declaration否
特殊视觉效果第二层component_declaration + theme否
复杂交互面板第三层artifact_patch受限

Artifact 生命周期

1
创建 — LLM 首次输出 artifact_patch（patch_type: "full"），Runtime 校验权限后写入 artifacts 表。
2
版本更新 — 后续 patch 引用同一 artifact_id，基于上一版本应用 patch 生成新版本。
3
离屏卸载 — 滚出视口后保存静态 HTML 快照，销毁 iframe 释放资源。
4
恢复 — 滚回视口后先展示快照，后台重建 iframe 并恢复状态。
5
销毁 — 会话删除或手动移除时软删除。

UI Schema 定义

State Diff 
{
  "type": "state_diff",
  "changes": {
    "items.add": [{ "id": "key", "name": "钥匙" }],
    "character.player.hp": { "from": 80, "to": 65 }
  }
}
Component Declaration 
{
  "type": "component_declaration",
  "renderer": "item_card",
  "theme": "dark_fantasy",
  "props": {
    "name": "血月碎片",
    "rarity": "legendary"
  }
}
Artifact Patch 
{
  "type": "artifact_patch",
  "artifact_id": "combat_panel",
  "patch_type": "full",
  "payload": {
    "html": "<div>...</div>",
    "css": "...",
    "js": "..."
  }
}

iframe 沙箱规范

sandbox 属性

<iframe sandbox="allow-scripts allow-same-origin"
        referrer-policy="no-referrer"
        loading="lazy"></iframe>
  • 不允许 allow-top-navigation — 阻止 iframe 跳转主页面。
  • 不允许 allow-popups — 阻止弹窗。
  • 不允许 allow-forms — 阻止表单提交。

postMessage 通信协议

主页面 → iframe
消息类型说明
init初始状态和 props。
state_update状态更新。
patch增量 patch。
resize容器尺寸变化。
visibility可见性变化。
iframe → 主页面
消息类型说明
readyiframe 加载完成。
state_change用户交互导致的状态变化。
resize_request请求调整容器大小。
erroriframe 内错误报告。

API可用说明

资源预算

资源限制超限行为
DOM 节点数1000截断渲染，显示提示。
JS 大小200KB拒绝加载，显示静态预览。
CSS 大小100KB拒绝加载。
HTML 大小500KB拒绝加载。
单次 JS 执行时间50ms暂停执行 1 秒。
连续无响应3 秒冻结 iframe，显示重启按钮。
内存50MB销毁 iframe，显示快照。
动画帧率30fps限制 requestAnimationFrame。
定时器数量10拒绝新定时器。

快照策略

触发条件

条件行为
artifact 滚出视口保存快照，卸载 iframe。
用户切换会话保存快照，卸载 iframe。
标签页进入后台保存快照，冻结 iframe。
资源预算超限保存快照，销毁 iframe。

恢复流程

先进入视口时先展示快照（即时可见），后台创建 iframe 加载最新版本，ready 后替换快照。

最大同时活跃 artifact

同时运行的 iframe 数量限制为 5 个。超过限制时，最久未交互的 artifact 自动转为快照。

风险

风险影响缓解措施
LLM 输出恶意 JS在 iframe 内执行恶意代码。sandbox + CSP + postMessage + 资源预算。
iframe 逃逸绕过沙箱。不使用 allow-top-navigation、allow-popups、allow-forms。
DOM 堆积导致卡顿长会话创建大量 iframe。虚拟列表 + 离屏卸载 + 最大活跃数限制。
状态 patch 冲突多节点同时 patch。artifact patch 通过 Runtime 串行化。
资源预算被绕过代码动态分配内存。预算检查在主页面侧执行。
快照质量丢失无法还原动画和交互。快照只用于过渡，停留后立即重建 iframe。

验收测试

测试场景通过标准
第一层 state diff 渲染内置组件正确显示物品、属性或任务变化。
第二层白名单渲染器校验 renderer 和 props，使用白名单组件渲染。
第二层非法渲染器拒绝渲染，回退到第一层或显示错误。
第三层 iframe 沙箱隔离无法访问 window.parent.document、localStorage、网络。
postMessage 通信主页面发送 init 和 state_update，iframe 正确接收和响应。
离屏卸载与恢复iframe 被卸载后恢复，用户无感知中断。
资源预算 DOM 超限超过 1000 节点时截断渲染。
资源预算 JS 超限超过 200KB 时拒绝加载。
最大同时活跃数超过 5 个时自动转为快照。
多节点 patch 同一 artifactpatch 按序应用，不出现覆盖冲突。
长会话 500 轮后性能聊天滚动流畅。
LLM 无法注入主 DOM所有层的输出都不直接插入主聊天 DOM。
