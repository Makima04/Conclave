内容包规范

定义平台四种内容包（Character Pack、World Pack、Agent Graph Pack、Plugin Pack）的结构、manifest schema、资源约定、版本规则、导入导出流程和校验机制。

文档中心 架构首页 通用规范 角色包 世界包 行为包 能力包 导入导出 SillyTavern 迁移 风险 验收测试

目标

  • 为四种内容包提供统一的结构和校验规范，让创作者、导入工具和 Runtime 都能以相同方式处理内容包。
  • 普通创作者可以通过 Character Pack 和 World Pack 快速开始 RP，不需要理解 Agent 图。
  • 高级创作者可以通过 Agent Graph Pack 自定义编排逻辑。
  • SillyTavern 角色卡、世界书和预设有明确的迁移路径。
  • 所有内容包可导入、可导出、可校验、可版本管理。

通用规范

Manifest Schema

每个内容包的根目录必须包含 manifest.json。

{
  "pack_type": "character",
  "id": "char_archivist",
  "name": "档案管理员",
  "version": "1.2.0",
  "author": "creator_name",
  "description": "旧档案馆的守护者，知晓古老文献和血月仪式的部分真相。",
  "platform_version": ">=0.1.0",
  "dependencies": [],
  "assets": ["assets/portrait.png", "assets/voice_sample.mp3"]
}
字段类型必填说明
pack_typestring是character、world、graph、plugin。
idstring是全局唯一标识，只允许 [a-z0-9_]。
namestring是人类可读名称。
versionstring是SemVer 格式。
authorstring否创作者标识。
descriptionstring否一句话描述。
platform_versionstring否兼容的平台版本范围，SemVer range 格式。
dependenciesstring[]否依赖的其他包 id。
assetsstring[]否包内资源文件路径列表。

版本规则

  • 使用 Semantic Versioning：MAJOR.MINOR.PATCH。
  • MAJOR：不兼容的结构变更。MINOR：向后兼容的功能新增。PATCH：向后兼容的修复。

资源文件约定

  • 所有资源放在 assets/ 目录下。
  • 图片：png、jpg、webp，单文件不超过 5MB。
  • 音频：mp3、ogg、wav，单文件不超过 20MB。
  • 字体：woff2，不超过 2MB。
  • 资源路径只允许引用包内文件，不允许绝对路径或 .. 逃逸。

Character Pack — 角色包

角色包定义一个 NPC 或用户角色的完整信息，是 RP 最基础的内容单元。

目录结构

character/
  manifest.json          # 包元数据，pack_type = "character"
  public.md              # 公开设定：外貌、身份、公开背景、说话风格
  hidden.md              # 隐藏设定：秘密、真实动机、隐藏知识
  personality.json       # PersonalityCore + 可演化 PersonalityState 初始值
  relationships.json     # 初始关系图
  greeting.md            # 开场白
  example_dialogues.md   # 示例对话（可选）
  assets/                # 头像、语音、背景图等资源

personality.json

{
  "core": {
    "values": ["knowledge_preservation", "loyalty_to_institution"],
    "fears": ["loss_of_memory", "being_forgotten"],
    "desires": ["uncover_blood_moon_truth"],
    "speech_style": "正式、偶尔引用古籍，语速缓慢",
    "knowledge_boundary": ["archive_layout", "ancient_scripts"],
    "ooc_redlines": ["sudden_betrayal_without_cause", "casual_violence"]
  },
  "initial_state": {
    "mood": "calm_vigilant",
    "trust_player": 0.3,
    "short_term_goals": ["assess_player_intent"],
    "behavioral_tendencies": ["cautious_but_polite"]
  }
}

公开与隐藏设定分离

  • public.md：NPC 自己知道且愿意分享的信息。
  • hidden.md：NPC 不会主动透露的秘密、真实动机、隐藏知识边界。
  • Runtime 构建 ContextBundle 时，public.md 全量注入，hidden.md 按条件注入。
  • Director 和 Consistency Agent 可以读取 hidden.md。

World Pack — 世界包

世界包定义 RP 世界的地点、势力、规则、时间线和导演强度配置。

目录结构

world/
  manifest.json          # 包元数据，pack_type = "world"
  lore/
    public/              # 公开世界知识：地理、历史、文化
    hidden/              # 隐藏世界知识：未揭示的真相、幕后势力
  timeline.json          # 世界时间线
  rules.json             # 世界规则：能力体系、物理规则
  director_config.json   # 导演强度和用户权限配置
  assets/                # 地图、势力徽章

rules.json 示例

{
  "power_system": {
    "type": "knowledge_based",
    "rules": ["力量来源于对古老文献的理解深度"]
  },
  "hard_constraints": [
    "已死角色不能复活，除非世界规则明确允许",
    "时间不能倒流",
    "角色不能出现在两个地点"
  ]
}

director_config.json

字段说明
director_strengthpassive（用户主导）、collaborative（协作）、strict（导演主导）。
user_narrative_authorityfull、character_actions、limited。
npc_autonomylow（被动响应）、moderate（有独立行为）、high（主动推动剧情）。

Agent Graph Pack — 行为包

Agent Graph Pack 定义自定义节点图，让高级创作者控制编排逻辑。参照 Agent Runtime。

目录结构

graph/
  manifest.json          # 包元数据，pack_type = "graph"
  graph.json             # 节点图定义
  prompts/               # 自定义 prompt 模板
  config.json            # 运行时配置：循环限制、token 预算、超时
  assets/

graph.json 示例

{
  "nodes": [
    { "id": "director", "type": "DirectorNode", "config": {} },
    { "id": "npc_archivist", "type": "NpcNode", "config": { "character_id": "archivist" } },
    { "id": "writer", "type": "WriterNode", "config": {} },
    { "id": "memory", "type": "MemoryNode", "config": {} }
  ],
  "edges": [
    { "from": "director", "to": "npc_archivist", "type": "sequence" },
    { "from": "npc_archivist", "to": "writer", "type": "sequence" },
    { "from": "writer", "to": "memory", "type": "sequence" }
  ],
  "entry": "director",
  "max_loop_count": 2,
  "max_total_nodes": 8
}

权限声明

权限说明
read_public_lore读取公开世界知识。
read_hidden_lore读取隐藏世界知识，需用户授权。
write_memory提交记忆候选变更。
custom_prompts使用自定义 prompt 模板。
parallel_execution启用并行节点。

Plugin Pack — 能力包

Plugin Pack 为平台扩展新节点类型、工具、检索器或渲染组件。V1 只做高层定义，详细规范留待插件系统规范（待补）。

目录结构

plugin/
  manifest.json          # 包元数据，pack_type = "plugin"
  permissions.json       # 权限 manifest
  nodes/                 # 自定义节点实现
  tools/                 # 工具定义
  assets/

安全边界

  • Plugin Pack 默认没有读取隐藏设定、网络访问、文件访问或写入记忆的权限。
  • 所有权限必须在 permissions.json 中显式声明。
  • 用户导入时看到权限列表并逐项授权。
  • 运行时由平台强制执行权限，插件代码节点在沙箱内运行。

导入流程

1
解压 ZIP — 检查 ZIP 结构是否合法，不含路径逃逸（../）。
2
Manifest 校验 — 必填字段完整、id 格式合法、version 是有效 SemVer。
3
平台兼容性 — 检查 platform_version 是否满足。
4
Schema 校验 — 按 pack_type 检查必需文件是否存在，JSON 格式正确，Markdown 非空。
5
依赖检查 — dependencies 中的包是否已导入。
6
权限声明 — Plugin Pack 和 Agent Graph Pack 的权限是否被用户授权。
7
资源完整性 — assets 声明的文件全部存在，大小和格式符合约定。
8
写入 — 写入 content_packs 表，解压到 storage_path，返回导入报告。

校验错误分级

级别行为示例
error阻止导入manifest 缺少必填字段、依赖包未导入、权限未授权。
warning允许导入但提示资源文件偏大、版本过旧、缺少可选文件。

导出流程

  • 读取 content_packs 表和 storage_path，校验内部一致性，打包为 ZIP。
  • 以 <id>_v<version>.zip 命名返回。
  • 导出不包含运行时数据（trace、state、memory），只包含创作者原始内容。

SillyTavern 迁移映射

角色卡 → Character Pack
  • description → public.md
  • personality → personality.json core
  • mes_example → example_dialogues.md
  • first_mes → greeting.md
  • system_prompt → personality.json system_prompt
世界书 → World Pack
  • 公开条目 → lore/public/*.md
  • 隐藏/选择性插入条目 → lore/hidden/*.md
  • 触发条件 → 条目 metadata

迁移风险提示

  • 公开/隐藏设定划分：无法可靠自动分离时，全部放入 public.md，提示用户手动整理 hidden.md。
  • 世界书选择性插入逻辑与平台 ContextBundle 可见性机制不完全对应。
  • SillyTavern 扩展脚本和自定义 CSS 需通过 Plugin Pack / Artifact Renderer 重新实现。

风险

风险影响缓解措施
公开/隐藏设定划分不当NPC 泄露不该知道的秘密。导入后提供可见性预览工具。
资源文件体积失控导入包过大。资源大小限制 + 导入时警告 + 运行时懒加载。
Agent Graph Pack prompt 注入恶意 prompt 可能绕过权限。prompt 只在节点自己的 ContextBundle 内使用，输出经过 Runtime 校验。
版本兼容性断裂新旧版本不兼容。manifest 声明 platform_version，导入时严格检查。
SillyTavern 迁移信息丢失自动迁移不完美。迁移报告列出无法自动处理的部分。
依赖包缺失引用的包未导入。导入时检查依赖，缺失时阻止导入。

测试场景通过标准
