定义项目文档的同步矩阵、更新触发条件、Markdown/HTML 一致性要求和自动化检查建议。防止实现进度与文档描述长期漂移。

目标

  • 项目实际代码、目录结构、Markdown 文档、HTML 文档和 AGENT.md 五者始终描述同一状态。
  • 任何变更都有明确的文档更新触发规则，不依赖开发者记忆。
  • 新增文档时有标准化流程，不会遗漏索引更新。
  • 文档内链接始终有效，不存在死链。

同步层次

代码 ↔ 文档

代码变更后，对应文档描述必须反映实际行为。

Markdown ↔ HTML

内容完全一致，HTML 是 Markdown 的视觉化呈现。

文档 ↔ 索引

文档中心和架构首页列出所有已存在文档。

索引 ↔ AGENT.md

AGENT.md 的项目结构索引、阅读顺序和待补规范与实际状态一致。

文档变更触发

变更内容必须同步
新增文档创建 .md 和 .html → 更新 docs.md + HTML → 更新 index.md + HTML → 更新 AGENT.md
删除文档反向操作，所有引用必须清除。
重命名文档删除 + 新增，所有交叉引用更新。
文档内容变更同步更新对应的 HTML 版本。

Markdown ↔ HTML 同步规则

一对一映射

每个 docs/*.md 文件对应一个 docs/html/*.html 文件。HTML 版本必须包含与 Markdown 相同的信息段落、表格、代码块和交叉引用。

同步时机

  • 先 Markdown 后 HTML：修改内容时先编辑 Markdown，再同步 HTML。
  • 同时提交：Markdown 和 HTML 的变更在同一个 commit 中提交。
  • 内容一致：差异只在视觉样式（CSS），信息内容完全相同。

HTML 模板规范

  • 相同的 CSS 变量（--bg、--panel、--ink、--accent 等）。
  • 相同的布局结构：header → nav（sticky）→ main → section.band。
  • 响应式断点：max-width: 900px。
  • 导航栏链接到相关文档（HTML 版本之间互链，不链到 Markdown）。

索引文件维护

docs.md / docs.html

文档中心是所有文档的入口索引。当前文档列出所有 docs/*.md 文件。待补文档列出计划中但尚未创建的文档。建议阅读顺序包含所有当前文档。

index.md / index.html

架构首页是项目总览。新增文档时检查是否需要在总览中添加引用。不承载过深细节。

AGENT.md

LLM 编程 Agent 的协作协议。触发更新：新增文档 → 阅读顺序；新增目录 → 项目结构索引；完成待补文档 → 待补规范列表。

自动化建议

文档结构检查

检查 docs/*.md 和 docs/html/*.html 是否一一对应。检查 docs.md 中列出的文档是否全部存在。

链接检查

检查所有 Markdown 和 HTML 文件中的链接是否指向存在的文件。检查锚点链接是否在目标文件中存在。

同时修改检查

如果 commit 修改了 docs/foo.md 但未修改 docs/html/foo.html，发出警告。反向亦然。

AGENT.md 一致性检查

检查 AGENT.md 项目结构索引和阅读顺序中的文件是否全部存在。检查待补规范列表中是否有文档已创建。

文档生命周期

新增文档流程

1
确认文档属于哪个模块。
2
在 docs.md 的"待补文档"中确认已登记。
3
编写 docs/<name>.md，包含：标题、摘要、标签、导航链接、目标、关键设计、风险、验收测试。
4
编写 docs/html/<name>.html，遵循 HTML 模板规范。
5
更新 docs/docs.md 和 docs/html/docs.html。
6
更新 docs/index.md 和 docs/html/index.html。
7
更新 AGENT.md：阅读顺序、项目结构索引、模块职责边界、待补规范。
8
检查所有新文件中的链接是否正确，单独 commit。

修改文档流程

1
读取目标文档和相关文档，确认变更范围。
2
编辑 Markdown，同步编辑 HTML。
3
如变更影响其他文档、索引或 AGENT.md，同步更新。
4
运行链接检查，commit。

风险

风险影响缓解措施
同步规则本身过时规则描述的文件结构与实际不符。每次新增文档时检查映射表。
HTML 维护成本高手写 HTML 容易出错。后续可引入自动转换工具。
AGENT.md 更新遗漏LLM Agent 读到过时信息。CI 自动检查一致性。
交叉引用断裂链接指向已删除文件。CI 链接检查。
阅读顺序混乱新文档未在阅读顺序中。新增文档流程强制更新阅读顺序。

验收测试

测试场景通过标准
docs/*.md 与 docs/html/*.html 一一对应每个 Markdown 有对应 HTML，无多余或缺失。
docs.md 当前文档列表与实际文件一致列出的所有文档在目录中存在，反向亦然。
AGENT.md 阅读顺序文件全部存在每个路径都指向实际存在的文件。
AGENT.md 待补规范与 docs.md 待补文档一致两个列表描述同一组待补文档。
所有文档内链接有效无死链。
新增文档后所有索引已更新docs.md、index.html、AGENT.md 均反映新文档。
Markdown 和 HTML 同一 commit 提交不出现只改 Markdown 不改 HTML 的 commit。
CI 检查通过结构检查、链接检查、同时修改检查均无错误。
