建议阅读顺序

顺序 文档 为什么先读
1 项目总架构 先明确项目不是普通聊天应用，而是长篇叙事和多 Agent 编排平台。
2 长期记忆与叙事一致性 这是平台最核心的问题域，决定后端、Agent、数据库和测试方式。
3 Agent Runtime 规范 先明确多 Agent 每轮如何执行、图如何分级、状态如何提交。
4 Agent 边界与权限 多 Agent 是否有效，取决于上下文隔离、权限和写入控制。
5 技术选型 最后再看技术栈，确保技术选择服务架构目标。
6 数据库与 API 规范 进入实现前先明确数据模型和 API 契约。
7 内容包规范 创作者入口依赖内容包，SillyTavern 迁移也依赖它。
8 Artifact Renderer 规范 防止 LLM 生成代码污染 DOM 或拖垮长会话性能。
9 测试与评测规范 平台核心风险必须能回归验证。
10 项目实现优先级 进入实现前确认先做哪些契约、MVP 和核心 Runtime 能力。
11 文档同步规则 理解文档维护规则，防止实现与文档长期漂移。

维护规则

  • index.html 只放总览和入口，不承载过深细节。
  • 长期一致性、Agent 边界、内容包、Artifact、插件、API 分别独立成文。
  • 每份文档都必须包含：目标、关键设计、风险、验收测试。
  • 技术选型必须服从长期记忆与 Agent 边界，而不是反过来。
  • 后续进入实现前，先把数据模型和 API 文档补齐。
  • 正式文档必须保持 Markdown 与 HTML 版本同步，文档入口和项目进度说明也必须同步。
