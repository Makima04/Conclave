目标

  • 平台每个核心风险域都有可自动回归的测试场景。
  • 测试不依赖真实 LLM API，使用 Mock Provider 回放预设响应。
  • 长会话测试可自动模拟 100-1000 轮对话。
  • 每个验收测试能明确判定通过或失败，不依赖主观评估。
  • 性能基准有具体数值目标。

测试分类

单元测试

纯逻辑验证

数据模型、schema 校验、state diff 合并、patch 应用、权限检查。不涉及 LLM 调用或网络。

工具：Rust #[test]，前端 Vitest。

集成测试

API 到数据库完整链路

使用真实 SQLite 内存数据库，Mock LLM Provider。覆盖 API 端点、SSE 流式输出、内容包导入。

工具：Rust axum::test + sqlx 内存模式。

场景测试

端到端用户场景

使用 Mock LLM Provider 回放多轮预设响应，覆盖八大风险域。

性能基准

延迟和资源占用

关键路径的延迟、内存和渲染性能。CI 中每次 PR 运行。

工具：Rust criterion，前端 Lighthouse。

八大风险域测试场景

域 1：长会话回归

测试通过标准
100 轮后召回第 10 轮事件从事件账本或结构化状态正确召回。
500 轮后伏笔仍可检索伏笔状态仍为 open 或 hinted，被正确召回。
1000 轮后状态一致性结构化状态与事件账本一致，无丢失或矛盾。
摘要不累积错误后续摘要不引入与原始事件矛盾的事实。
长会话内存占用1000 轮后单会话内存增量不超过 200MB。

域 2：OOC 检测

测试通过标准
角色不泄露未知秘密NPC 输出不包含 knowledge_boundary 外的秘密。
关系不跳变信任值变化不超过合理幅度，有触发事件支撑。
能力不膨胀WorldJudge 拒绝未设定能力。
人格核心不被冲淡200 轮后 personality_core 未被修改。
成长有据可循growth_arc 有阶段推进记录和事件引用。

域 3：伏笔回收

测试通过标准
伏笔种下后可检索出现在 foreshadowing 表，状态为 open。
伏笔在合适场景被提醒Director PlanResult 包含伏笔引用。
伏笔回收后状态更新状态变为 resolved。
伏笔长期未回收有提示100 轮未处理时向 Director 发出提醒。
多伏笔不混淆只有匹配条件的伏笔被召回。

域 4：Agent 越权

测试通过标准
NPC 无法读取导演计划ContextBundle 不含 director_plan。
NPC 无法读取未知秘密ContextBundle 不含未被告知的秘密。
Writer 无法发明事实Consistency Agent 检测到冲突。
插件无法读取隐藏世界书无 read_hidden_lore 权限时 ContextBundle 不含隐藏内容。
插件无法越权写入无 write_memory 权限时 proposal 被拒绝。
handoff 不传递完整推理Runtime 拦截，只传递结构化结果。

域 5：循环与成本

测试通过标准
超过 max_loop_count循环被阻止，走 fallback。
总节点数超过限制后续节点被阻止。
Token 预算超限中止后续节点执行。
运行时间超限中止执行，记录 trace。
并发节点数超限只同时执行限制数量的节点。

域 6：Artifact 隔离

测试通过标准
iframe 无法访问主 DOMwindow.parent.document 抛出安全错误。
iframe 无法访问存储localStorage.setItem 被阻止。
iframe 无法发起网络请求fetch 被 CSP 阻止。
DOM 节点超限超过 1000 节点时截断渲染。
离屏卸载与恢复iframe 被卸载，恢复后状态一致。
最大活跃数超过 5 个时自动转为快照。

域 7：状态提交安全

测试通过标准
low 风险自动提交committed_by = "runtime"。
medium 风险需 Director 审核committed_by = "director"。
high 风险需证据链必须附带 ChangeJustification。
高风险不自动提交proposal 被拒绝或挂起。
状态版本递增version 每次递增，不跳号不重复。

域 8：流式输出

测试通过标准
完整轮次事件序列按序收到 turn_start → ... → turn_end。
流式文本增量推送增量拼接后等于最终消息内容。
连接中断恢复可通过 GET /messages 拉取已完成消息。
大消息分块分多个事件推送，无丢失或乱序。
并发会话隔离各自 SSE 流互不干扰。

测试工具与基础设施

Mock LLM Provider

回放预设响应序列，实现与真实 Provider 相同的 Adapter 接口。支持固定响应、序列响应、条件响应、流式输出模拟、延迟和错误模拟。

测试用内容包 Fixtures

预定义的 Character Pack、World Pack 和 Agent Graph Pack，覆盖标准 NPC、简单 NPC、复杂 NPC、标准世界、简单世界、多种图配置。

长会话生成器

按预定义场景自动选择每轮用户输入，每 N 轮自动运行断言检查，最终输出一致性报告。支持 100-1000 轮。

风险

风险影响缓解措施
Mock Provider 不代表真实 LLM 行为测试通过但真实场景失败。Mock 只测系统逻辑；LLM 质量用独立评测集。
长会话测试耗时过长CI 时间不可接受。日常 CI 只跑 100 轮；500+ 轮在每日任务。
OOC 检测依赖主观标准测试结果不可靠。只检查结构化约束，不评估文学质量。
测试 fixture 维护成本格式变更后 fixture 失效。fixture 和 schema 共用 JSON Schema 定义。
性能基准环境差异CI 和本地性能不同。关注相对回归百分比，不关注绝对值。
