角色卡渲染架构迁移：postMessage Bridge → ST 同源直连运行时

 Context（为什么做这件事）

 项目渲染 SillyTavern 前端角色卡反复出 bug——修好一张卡另一张又坏（git 历史可证：7 类反复 bug，9309215/3141f44/f8a841c/09ece68/6fa0042 等连续修卡提交）。根因不是单个 bug，而是架构与参照实现（SillyTavern +
 酒馆助手 JS-Slash-Runner）存在三个根本性错位：

 目标架构

 宿主页面 (React)
 ├── window 全局: _ (lodash) / $ (jQuery) / YAML / toastr(facade) / z (zod) / showdown
 │   ├── window.TavernHelper  — 完整 API 对象 + _bind 模式（同步变量读取）
 │   └── window.SillyTavern   — context 代理
 ├── StRuntimeStore — 会话内存态唯一事实源
 │   ├── chat: ST 形状消息数组 (message_id/name/role/swipes/swipe_id/variables[swipe_id]/data)
 │   ├── 变量 scopes: chat(后端 session_variables) / message(消息 metadata) / global+character(localStorage)
 │   ├── regex 脚本 / 角色数据
 │   └── 写 = 同步改内存 + emit 事件 + debounced 持久化到现有 REST
 ├── eventSource — tavern_events 全集；SSE/turn 生命周期 → GENERATION_STARTED/ENDED、MESSAGE_RECEIVED 等
 ├── 脚本 iframe × N (TH-script--{name}--{id}, 常驻隐藏, 会话切换时销毁)
 └── 消息 iframe × N (TH-message--{messageId}--{n}, blob URL 同源)
     head 顺序: meta → base → reset style → CDN 库 → predefine.js → adjust_viewport.js → adjust_iframe_height.js

 新代码集中在 frontend/src/pages/st-runtime/ 目录。
